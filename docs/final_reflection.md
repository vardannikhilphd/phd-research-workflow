
## Answer the following questions

1. How does Git help in maintaining research traceability?
- Ans.1 Git helps maintain research traceability by recording every change in code, data, and documents with timestamps, authorship, and explanations, enabling researchers to track the complete history of their work, manage different experimental versions, collaborate transparently, and reliably reproduce results at any stage of the research process.
2. Why is GitHub useful for academic collaboration?
- Ans.2 GitHub facilitates academic collaboration by providing a platform for sharing code, tracking issues, managing projects, and enabling code reviews through pull requests. It allows researchers to work together on the same codebase, contribute to open-source projects, and maintain transparency in their research process.
3. Why should researchers create branches instead of working directly on main?
- Ans.3 Creating branches allows researchers to work on new features, experiments, or fixes without affecting the stable main branch. This prevents introducing bugs or incomplete changes into the primary codebase, enables parallel development, and makes it easier to review and merge changes later.
4. What is the role of Pull Requests in collaborative work?
- Ans.4 Pull Requests (PRs) allow team members to propose changes to the codebase, review the proposed modifications, discuss improvements, and approve or request changes before merging. This ensures code quality, facilitates knowledge sharing, and maintains a record of discussions and decisions.
5. Why are reproducibility files important in research repositories?
- Ans.5 Reproducibility files, such as requirements.txt, environment.yml, or Dockerfiles, ensure that others can recreate the exact computational environment used in the research. This is crucial for verifying results, building upon previous work, and maintaining scientific integrity.
6. What is the purpose of creating a version tag such as v1.0.0?
- Ans.6 Version tags mark specific points in the repository's history as stable releases. They allow researchers to reference exact versions of their code and data associated with published results, enabling precise reproduction and citation of research artifacts.
7. Why should a repository include a citation file?
- Ans.7 A citation file (like CITATION.cff or CITATION.md) provides standardized metadata for citing the repository in academic papers. It helps other researchers properly credit the work and makes it easier to reference the code and data in publications.
8. How does this semester’s Git/GitHub workflow support better PhD research practice?
- Ans.8 This workflow supports better PhD research practice by promoting version control, collaboration, documentation, and reproducibility. It encourages systematic tracking of changes, transparent peer review processes, and the creation of shareable, citable research artifacts that enhance the credibility and impact of the research.



## Further, Answer the following:

9. Why is reproducibility important in PhD-level research?
- Ans.9 Reproducibility is crucial in PhD-level research because it allows other scientists to verify results, build upon previous work, and ensure the integrity of scientific findings. It prevents errors, promotes transparency, and is essential for advancing knowledge in a field.

10. What is the difference between a commit, a Pull Request, a tag, and a release?
- Ans.10 - A commit is a snapshot of changes to the repository at a specific point in time.
- A Pull Request is a proposal to merge changes from one branch into another, allowing for review and discussion.
- A tag is a reference to a specific commit, often used to mark versions.
- A release is a snapshot of the repository at a particular tag, often including additional metadata and downloadable assets.

11. Why should code, data, and documentation be version-controlled together?
- Ans.11 Version-controlling code, data, and documentation together ensures that all components of the research are synchronized and traceable. This allows researchers to recreate exact states of their work, maintain consistency between code and results, and provide complete, reproducible research packages.

12. What problems arise if research repositories are poorly documented?
- Ans.12 Poor documentation can lead to confusion, difficulty in reproducing results, wasted time trying to understand code or data, increased risk of errors, and reduced collaboration potential. It can also make it harder to build upon the work or get proper credit for contributions.

13. How can GitHub support thesis work, publications, and collaborative research projects?
- Ans.13 GitHub supports thesis work by providing version control for code and documents, issue tracking for research tasks, and a platform for sharing work. For publications, it allows archiving code and data associated with papers. For collaborative projects, it enables team coordination, code reviews, and open-source contributions.
